copyfail / afalg-defense

Userspace defensive primitives for CVE-2026-31431 ("Copy Fail") and the broader AF_ALG attack surface. LD_PRELOAD shim plus a comprehensive read-only host posture auditor. Source: github.com/rfxn/copyfail.

Add the repo (EL8 / EL9 / EL10)

A single copyfail.repo file works for all three: it uses $releasever + $basearch substitution that dnf expands at fetch time. 

curl -sSL https://rfxn.github.io/copyfail/copyfail.repo \
  | sudo tee /etc/yum.repos.d/copyfail.repo
sudo dnf install -y afalg-defense

After install, the LD_PRELOAD shim is on disk but not yet active. Wiring /etc/ld.so.preload from a package post-install is too dangerous - one broken upgrade locks every dynamic-linked binary out of dlopen. Activation is an explicit operator action: 

sudo /usr/sbin/copyfail-shim-enable    # smoke-tests, then writes /etc/ld.so.preload
sudo /usr/sbin/copyfail-shim-disable   # reverses it

Verify the block is live: 

python3 -c 'import socket; socket.socket(socket.AF_ALG, socket.SOCK_SEQPACKET, 0)'
# expect: PermissionError [Errno 1] Operation not permitted

Subpackages

PackageArchContents
afalg-defensex86_64Meta - pulls shim + auditor
afalg-defense-shimx86_64 /usr/lib64/no-afalg.so + copyfail-shim-{enable,disable} helpers
afalg-defense-auditornoarch /usr/sbin/copyfail-local-check (Python, stdlib-only, read-only)

Auditor-only is a valid install pattern for hosts that do not yet trust an LD_PRELOAD on every dynamic-linked process: 

sudo dnf install -y afalg-defense-auditor

Direct downloads

Each release on github.com/rfxn/copyfail/releases ships per-EL binary RPMs and an SRPM:

ELRepository tree
8 /repo/8/x86_64/
9 /repo/9/x86_64/
10 /repo/10/x86_64/
Unsigned in 1.0. The published RPMs are not yet GPG-signed. The copyfail.repo file therefore sets gpgcheck=0. Operators with hardened policy should rebuild from the published SRPM under their own signing infrastructure, or verify by sha256 against the release-asset checksums.

Defense-in-depth context

The shim is one rung of a five-layer ladder (kernel patch > modprobe blacklist > systemd RestrictAddressFamilies > LD_PRELOAD shim > seccomp). The auditor scores all five against the running host. See the README for the full ladder, what each layer fails to catch, and why they ship together.
